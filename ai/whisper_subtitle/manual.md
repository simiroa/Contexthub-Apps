# Whisper Subtitle AI (Qt)

## 개요
Whisper 기반 자막 생성 파이프라인을 큐 기반으로 실행하고, 결과를 바로 검수/수정할 수 있는 앱입니다.

입력 큐의 미디어를 선택하면 미리보기가 동기화되고, 생성/편집이 끝난 세그먼트는 파일별 세션으로 저장되어 재시작 시 이어서 작업할 수 있습니다.

## 핵심 기능
- 큐 편집: 여러 파일 등록, 선택 파일 전환, 배치 실행/단일 실행
- 세그먼트 편집: `ID / 시작 / 종료 / 텍스트` 표 편집
- 원문 편집: SRT 또는 VTT 텍스트 탭을 통해 빠른 보정
- 파싱 안정성: 원문 텍스트 파싱 실패 시 기존 세그먼트 보존
- 세션 저장: `<미디어명>.subtitle_session.json`로 자막 상태와 옵션 기록
- 플레이어 동기화: 현재 재생 위치 기반으로 해당 세그먼트 하이라이트
- 기본 출력 폴더: 입력 파일 폴더, 필요 시 사용자 지정 출력 경로 사용

## 사용법
1. 앱을 실행하고 `+` 버튼으로 미디어를 추가합니다.
2. 좌측 큐에서 파일을 선택하면 미리보기와 편집 대상이 전환됩니다.
3. 우측에서 모델/태스크/디바이스/언어를 지정하고 출력 포맷(SRT/VTT/TXT/JSON)을 선택합니다.
4. 실행 모드 선택:
   - `선택 대상만` 또는 `Batch`
5. `Run`으로 생성 실행.
6. 생성 후:
   - 좌측/중앙 편집기에서 세그먼트 보정
   - Raw 탭에서 SRT 또는 VTT 텍스트를 직접 수정 후 `Apply`
   - 출력 폴더 Reveal 또는 세션 export로 저장 상태 확인

## 자막 파일
- 기본적으로 SRT/VTT가 생성됩니다.
- TXT/JSON은 선택한 포맷만 추가로 생성됩니다.
- TXT는 `segment.text` 줄 단위 병합, JSON은 세그먼트 객체 배열입니다.

## 세션
- `subtitle_session.json`은 실행/편집 상태를 복구하는 기본 단위입니다.
- 앱을 다시 열어 같은 미디어를 선택하면 저장된 세션을 자동 로드해 이어서 편집할 수 있습니다.

## 권장 사용
- 긴 영상을 먼저 1~2분 구간만 생성해 타임싱크를 확인한 뒤 전체 배치를 실행하세요.
- SRT/VTT 원문 탭에서 파싱 에러가 뜨면 입력 형식을 맞춰 다시 적용하면 복구됩니다.

## 시스템 요구사항
- Faster-Whisper 실행 환경
- CUDA 환경이면 `cuda`, 아니면 `cpu` 선택
